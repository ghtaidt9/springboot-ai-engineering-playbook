# Persistence Strategy

## Scope
All services. Defines ORM choice, entity conventions, query patterns, and migration rules.
**Single source of truth for persistence decisions** (see also ADR-002).

---

## Rules

MUST:
- Use **Spring Data JPA** as the only ORM — no MyBatis, no raw JDBC for domain queries
- JPA entities live in `adapter/out/persistence/entity/` — never in `domain/`
- UUID primary keys generated by `@UuidGenerator`
- Audit fields (`createdAt`, `updatedAt`) via `@EnableJpaAuditing`
- All schema changes delivered via Flyway migration script before code change

MUST NOT:
- Use native SQL queries (`nativeQuery = true`) — use JPQL or Criteria API
- Reference JPA entities from domain or application layers
- Deploy a schema change without a corresponding migration file
- Edit or delete a committed Flyway migration (create a new one instead)

---

## Examples

**JPA entity (correct structure):**
```java
@Entity
@Table(name = "orders")
@EntityListeners(AuditingEntityListener.class)
public class OrderJpaEntity {

    @Id
    @UuidGenerator
    private UUID id;

    @Column(name = "customer_id", nullable = false)
    private String customerId;

    @Enumerated(EnumType.STRING)
    @Column(nullable = false, length = 20)
    private OrderStatus status;

    @OneToMany(cascade = CascadeType.ALL, orphanRemoval = true, fetch = FetchType.LAZY)
    @JoinColumn(name = "order_id")
    private List<OrderItemJpaEntity> items = new ArrayList<>();

    @CreatedDate
    @Column(updatable = false)
    private Instant createdAt;

    @LastModifiedDate
    private Instant updatedAt;
}
```

**Flyway migration:**
```sql
-- V001__create_orders_table.sql
CREATE TABLE orders (
    id          UUID         PRIMARY KEY,
    customer_id VARCHAR(100) NOT NULL,
    status      VARCHAR(20)  NOT NULL,
    created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_orders_customer_id ON orders(customer_id);
CREATE INDEX idx_orders_status ON orders(status);
```

## Query Guidelines

| Use case | Approach |
|----------|----------|
| Simple find by field | Spring Data method name (`findByCustomerId`) |
| Multi-field filter | Spring Data method name or `@Query` JPQL |
| Pagination | `Pageable` param + `Page<T>` return |
| Projection (read-only) | Interface or `record` projection |
| Reporting / aggregation | `@Query` JPQL with `GROUP BY` |
